Matching

  • Keyword

    Keyword queries are created by entering one or more keywords separated by whitespace. Results are ranked and returned based on how well they match the set of keyword query terms.

    Try it

  • Conditional

    Conditional operators may also be used in query terms to find objects. Attributes with string or boolean values may use the equivalence operator: =. Attributes with numerical or date values, may use the following operators: >, =, <.

    Try it

  • Advanced Boolean

    By default all queries entered into the search interface are treated as boolean AND queries. This default can be overrided by specifically declaring an OR query with the OR operator or. Boolean OR and AND queries can be combined using nesting with parantheses. Advanced boolean queries do not use partial matching and ranking.

    Try it

  • Link

    A query can also specify that linking operations be performed. To find all objects directly linked to a subset of objects, prepend the query which finds this subset with the link operator: link. Additional query terms can be placed to the left of this operator, to filter the type of linked objects desired.

    Try it

Projection and Aggregation Functions

  • Projection

    Additional functions may be performed on the returned data by providing a projection string. The simplest form of projection causes only requested attribute value pairs of cloud search objects to be returned. This is performed by adding a whitespace delimited list of the attributes, prepended with the projection operator: project. Partial matching and ranking is not performed on queries that contain a projection string.

    Try it

  • Aggregation

    Cloud search results may be aggregated using the aggregate operators using the following format: Y(x ..., Query) , where x is the attribute to be aggregated, Y is the aggregation function. Available aggregation functions are:

    • sum - sum
    • max - maximum
    • min - minimum
    • count - count
